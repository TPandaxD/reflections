When would you want to use a remote repository rather than keeping all your work
local?

  To share with other contributors or be able to access your work remotely.


Why might you want to always pull changes manually rather than having Git
automatically stay up-to-date with your remote repository?

  Because you may have changes that you don't want published yet, or you might be working on something that could get affected if you updated from remote


Describe the differences between forks, clones, and branches.  When would you
use one instead of another?

  Fork to steal someone else's code on GitHub, clone to copy a repository and branch to do different things in the same repository.


What is the benefit of having a copy of the last known state of the remote
stored locally?

  Because your remote repository may have been changed the last time you synced with it. By having a copy it allows git to track parent history of your files.
